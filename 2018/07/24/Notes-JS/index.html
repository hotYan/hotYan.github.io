null和undefined的区别？

NULL

1.null是一个表示”无”的对象，转为数值时为0;
2.null表示”没有对象”，即该处不应该有值;

  • 作为函数的参数，表示该函数的参数不是对象。
  • 作为对象原型链的终点。

undefined

1.undefined是一个表示”无”的原始值，转为数值时为NaN。

2.undefined表示”缺少值”，就是此处应该有一个值，但是还没有定义

  • 变量被声明了，但没有赋值时，就等于undefined。

  • 调用函数时，应该提供的参数没有提供，该参数等于undefined。

  • 对象没有赋值的属性，该属性的值为undefined。
  • 函数没有返回值时，默认返回undefined。

严格模式

进入”严格模式”的标志，老版本的浏览器会把它当作一行普通字符串，加以忽略

引用

  1. 将”use strict”放在脚本文件的第一行，则整个脚本都将以”严格模式”运行。
  2. 将”use strict”放在函数体的第一行，则整个函数以”严格模式”运行

因为第一种调用方法不利于文件合并，所以更好的做法是，借用第二种方法，将整个脚本文件放在一个立即执行的匿名函数之中

作用

  1. 消除Javascript语法的一些不合理、不严谨之处，减少一些怪异行为;

  2. 消除代码运行的一些不安全之处，保证代码运行的安全；

  3. 提高编译器效率，增加运行速度；

  4. 为未来新版本的Javascript做好铺垫。

hasOwnProperty()

  1. 执行对象查找时，永远不会去查找原型

  2. 用于指示一个对象自身(不包括原型链)是否具有指定名称的属性。如果有，返回true，否则返回false。

  3. 该方法属于Object对象，由于所有的对象都”继承”了Object的对象实例，因此几乎所有的实例对象都可以使用该方法。

、面向对象编程

1.面向对象编程思想

大概就是下面说的这样

面向对象是相对于面向过程的,比如你要充话费,你会想,下个支付宝,注册并登录,绑定银行卡,充值。这种种过程。但是对于你女朋友就不一样了,她是面向“对象”的,她会想,谁会充话费呢？当然是你了,她就给你电话,然后你把之前的做了一遍,然后她收到充值成功的短信,说了句,么么哒。这就是面向对象！女的思维大部分是面向“对象”的！她不关心处理的细节,只关心谁可以,和结果!

2.面向对象两个基本概念

类

类是对象的类型模板。例如:定义Student类来表示学生,类本身是一种类型,Student表示学生类型,但不表示任何具体的某个学生。

实例

实例是根据类创建的对象。例如:根据Student类可以创建出xiaoming、xiaohong、xiaojun等多个实例,每个实例表示一个具体的学生,他们全都属于Student类型。

3.面向对象编程特性

封装

把客观事物封装成抽象的类,并且类可以把自己的数据和方法只让可信的类或者对象操作,对不可信的进行信息隐藏。

继承

指这样一种能力:它可以使用现有类的所有功能,并在无需重新编写原来的类的情况下对这些功能进行扩展。

多态

允许将子类类型的指针赋值给父类类型的指针。实现多态:覆盖、重载。

、JS面向对象编程

JavaScript的面向对象编程和其它大多数语言如Java、C#的面向对象编程都不太一样。

JavaScript不区分类和实例的概念，而是通过原型（prototype）来实现面向对象编程。

JavaScript的原型链和Java的Class区别在于JS没有“Class”的概念，所有对象都是实例，所谓继承关系不过是把一个对象的原型指向另一个对象而已。

创建对象

Object构造函数

var hotyan = new Object();
hotyan.name = hotyan;
hotyan.age = 18;      //永远18...

对象字面量

var hotyan = {
  name : "hotyan",
  age : 18
};

属性名也可以使用字符串

var hotyan = {
  "name" : "hotyan",
  "age" : 18,
  5 : true       //数字属性名自动转换为字符串
};

访问对象属性【点方法/方括号表示法】

alert(hotyan.name);       //"hotyan"
alert(hotyan["name"]);    //"hotyan"

功能上没有区别,但方括号表示法可以通过变量来访问属性

var propertyName = "name";
alert(hotyan[propertyName]);    //"hotyan"

属性名使用的是关键字、保留字或包含会导致语法错误的字符、非字母非数字时可使用方括号表示法

hotyan["first name"] = "hotyan";    //"first name"包含一个空格,不能使用点方法

通常,除非必须使用变量来访问属性,否则建议用点方法。

1.原始模式

最简单的封装,把两个属性封装在一个对象里面。
缺点:

  1. 使用同一个接口创建很多对象(创建多个实例),会产生大量重复代码。
  2. 实例与原型之间,看不出任何联系。

2.工厂模式

工厂模式是广为人知的设计模式,抽象了创建具体对象的过程。
用函数来封装以特定接口创建对象的细节。

解决了创建多个相似对象的问题(即解决代码重复的问题)。
不足:

  1. 没有解决对象识别问题(即不知道对象的类型)
  2. 实例之间没有内在的联系,不能反映出它们是同一个原型对象的实例

3.构造函数模式

构造函数本身就是一个函数,出于创建新对象的目的而定义的。
内部使用了this变量,使用new运算符就能生成实例,并且this变量会绑定在实例对象上。
构造函数的函数名始终都应该以一个大写字母开头,非构造函数则应该以一个小写字母开头。

hotyan1和hotyan2都有一个constructor(构造函数)属性,该属性指向Person。

alert(hotyan1.constructor == Person);   //true
alert(hotyan2.constructor == Person);   //true

对象的constructor属性最初是用来标识对象类型的,但是提到检测对象类型,还是instanceof操作符更可靠一些。

alert(hotyan1 instanceof Object);   //true
alert(hotyan2 instanceof Object);   //true

alert(hotyan1 instanceof Person);   //true
alert(hotyan2 instanceof Person);   //true

创建的对象既是Object的实例,也是Person的实例。验证实例与原型之间的关系(即反映出它们是同一个原型对象的实例)。

将构造函数当作函数

构造函数与其他函数的唯一区别就是调用它们的方式不同。前面定义的Person()函数可以通过下列任何一种方式来调用。

//1.当作构造函数使用
var hotyan = new Person("hotyan",18,"Student");
hotyan.sayName();   //"hotyan"

//2.作为普通函数使用
Person("hotyan",18,"Student");  //添加到window
window.sayName();   //"hotyan"

//3.在另一个对象的作用域中调用
var O = new Object();
Person.call(O, "hotyan", 18, "Student");
O.sayName();   //"hotyan"

1.构造函数的典型用法,用new操作符来创建一个新对象。
2.不使用new操作符调用Person(),属性和方法都被添加window
3.使用call()或者apply()在某个特殊对象的作用域中调用Person()函数,O就拥有了所有属性和方法。

构造函数的问题

主要问题:
每个方法都要在每个实例上重新创建一遍。

不同实例上的同名函数是不相等的:

alert(hotyan1.sayName == hotyan2.sayName);  //false

尝试通过把函数定义转移到构造函数外部来解决问题:

sayName包含的是一个指向函数的指针,因此hotyan1和hotyan2对象就共享了在全局作用域中定义的同一个sayName()函数。确实解决了两个函数做同一件事情的问题,可是新问题又来了:

  1. 在全局作用域中定义的函数实际上只能被某个对象调用,这让全局作用域有点名不副实。
  2. 如果对象需要定义很多方法,就将定义很多个全局函数,使得自定义的引用类型丝毫没有封装性可言。

这些问题可以通过原型模式解决！

4.原型模式

我们创建的每一个构造函数都有一个prototype属性,这个属性是一个指针,指向一个原型对象。这个原型对象的所有属性和方法，都会被构造函数的实例继承。

不必在构造函数中定义对象实例的信息,直接将信息添加到原型对象中:

////////////后面将用...代替这部分代码/////////
function Person(){}

Person.prototype.name = "hotyan";
Person.prototype.age = 18;
Person.prototype.job = "Student";
Person.prototype.sayName = function(;{
    alert(this.name);
};
///////////到此处结束///////////////////////

仍然可以通过调用构造函数来创建新对象,新对象的属性和方法是由所有实例共享的。
即 hotyan1 和hotyan2 访问的都是同一组属性和同一个sayName()函数:

...
var hotyan1 = new Person();
hotyan1.sayName();  //"hotyan"

var hotyan2 = new Person();
hotyan1.sayName();  //"hotyan"

alert(hotyan1.sayName == hotyan2.sayName);  //true

要理解原型模式的工作原理必须先理解ECMAScript中原型对象的性质。

理解原型对象

默认情况下,所有原型对象都会自动获得一个constructor(构造函数)属性,这个属性是一个指针,指向prototype属性所在函数。
当调用构造函数创建一个新实例,内部包含一个内部属性[[Prototype]],这个属性是一个指针,指向构造函数的原型对象。

虽然在所有现实中都无法访问到[[Prototype]],可以通过isPrototypeOf()来确定对象之间是否存在这种关系:

alert(Person.prototype.isPrototypeOf(hotyan1));     //true
alert(Person.prototype.isPrototypeOf(hotyan2));     //true

ECMAScript 5新增Object.getPrototypeOf(),返回[[Prototype]]的值:

alert(Object.getPrototypeOf(hotyan1) == Person.prototype);  //true
alert(Object.getPrototypeOf(hotyan1).name);  //"hotyan"

属性屏蔽

在实例中添加name属性只会阻止我们访问原型中的同名属性,即使将name属性设置为null,也只会在实例中设置属性值为null,而不会恢复其指向原型的连接。必须使用delete操作符完全删除实例属性才能重新访问原型中的属性:

...
var hotyan1 = new Person();
var hotyan2 = new Person();

hotyan1.name = "hotyanhotyan";  //为对象实例添加name属性,将屏蔽原型对象中的同名属性。
alert(hotyan1.name);    //"hotyanhotyan" --来自实例
alert(hotyan2.name);    //"hotyan" --来自原型

delete hotyan1.name;
alert(hotyan1.name);    //"hotyan" --来自原型

使用hasOwnPrototype()方法可检测一个属行存在于实例还是原型。存在实例时返回true:

...
var hotyan1 = new Person();
var hotyan2 = new Person();

alert(hotyan1.hasOwnPrototype("name"));     //false

hotyan1.name = "hotyanhotyan";  //重写name属性
alert(hotyan1.name);            //"hotyanhotyan"--来自实例
alert(hotyan1.hasOwnPrototype("name"));     //true

alert(hotyan2.name);            //"hotyan"--来自原型
alert(hotyan2.hasOwnPrototype("name"));     //false

delete hotyan1.name;
alert(hotyan1.name);            //"hotyan"--来自原型
alert(hotyan1.hasOwnPrototype("name"));     //false

原型与in操作符

单独使用时,in操作符会在通过对象能够访问给定属性时返回true:

...
var hotyan1 = new Person();
var hotyan2 = new Person();

alert(hotyan1.hasOwnPrototype("name"));     //false
alert("name" in hotyan1);       //true

hotyan1.name = "hotyanhotyan";  //重写name属性
alert(hotyan1.name);            //"hotyanhotyan"--来自实例
alert(hotyan1.hasOwnPrototype("name"));     //true
alert("name" in hotyan1);       //true

alert(hotyan2.name);            //"hotyan"--来自原型
alert(hotyan2.hasOwnPrototype("name"));     //false
alert("name" in hotyan2);       //true

delete hotyan1.name;
alert(hotyan1.name);            //"hotyan"--来自原型
alert(hotyan1.hasOwnPrototype("name"));     //false
alert("name" in hotyan1);       //true

、 闭包

  1. 可以访问其他函数内部作用域的函数,在有函数嵌套的情况下,子函数用到了父函数的参数或者局部变量.

  2. 作用：具有封闭性，保护变量；避免污染全局变量

  3. 缺点：内存泄漏

、call、apply、bind的区别

1.call、apply

  • 改变函数体内部 this 的指向。
  • 作用完全一样，接受参数的方式不太一样
  • call:把参数按顺序传递进去
  • apply:把参数放在数组里。　　

2.call、apply、bind

  • apply 、 call 、bind 三者都是用来改变函数的this对象的指向的；
  • apply 、 call 、bind 三者第一个参数都是this要指向的对象，也就是想指定的上下文；
  • apply 、 call 、bind 三者都可以利用后续参数传参；
  • bind 是返回对应函数，便于稍后调用；apply 、call 则是立即调用 。

、原型链

  1. 对象的属性和方法，定义在自身或定义在它的原型对象上。由于原型本身也是对象，又有自己的原型，所以形成了一条原型链（prototype chain）。

  2. 如果一层层地上溯，所有对象的原型最终都可以上溯到Object.prototype，即Object构造函数的prototype属性指向的那个对象。而Object.prototype对象的原型就是没有任何属性和方法的null对象，而null对象没有自己的原型。

  3. 读取对象的某个属性时:

    JavaScript 引擎先寻找对象本身的属性,  
找不到?就到它的原型去找,  

还是找不到?就到原型的原型去找。  

直到最顶层的Object.prototype还是找不到?则返回undefined。

  4. 如果对象自身和它的原型，都定义了一个同名属性，那么优先读取对象自身的属性，这叫做“覆盖”（overriding）。

  5. 需要注意的是，一级级向上，在原型链寻找某个属性，对性能是有影响的。所寻找的属性在越上层的原型对象，对性能的影响越大。如果寻找某个不存在的属性，将会遍历整个原型链。
