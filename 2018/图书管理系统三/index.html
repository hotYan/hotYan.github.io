二、安装MySQL

  1. 终端下运行以下命令
    $ brew install mysql

    【暂时忽略你看到的一些提示！！！】

  2. 启动MySQL服务器

    先把服务器start再说

    $ brew services start mysql

    【现在可以跟着你看到的提示一步一步来！！！】

  3. 先安全登录MySQL

    运行提示一:

    $ mysql_secure_installation

    • 根据提示设置user root密码

      Enter password for user root：（此处输入你的密码）

    • 根据提示进行设置

      可一路yes

    • 设置完后，你会看到提示

      All done！
  4. 连接MySQL

    运行提示二:

    $ mysql -uroot

    或者

    $ mysql -u root -p

    回车后,输入刚刚设置的密码：

    Enter password：
  5. success！

    成功之后,就可以对Mysql进行操作啦

  6. 退出MySQL
    mysql> exit
Bye
  7. 关闭MySQL服务器
    $ brew services stop mysql
  8. 配置.bash_profile

    先查看你的mysql安装在哪:

    $ which mysql

    比如我返回的是:

    /usr/local/Cellar/mysql/5.7.22/bin/mysql

    打开.bash_profile文件:

    $ open ~/.bash_profile

    将下面这行代码加入.bash_profile文件:

    export PATH="/usr/local/Cellar/mysql/5.7.22/bin:$PATH"

    保存,退出后,保存新配置:

    $ source ~/.bash_profile
  9. 运行流程

    开启服务器:

    $ brew services start mysql

    登录Mysql:

    $ mysql -u root -p

    操作Mysql:

    【Mysql指令小结 】

    退出Mysql:

    mysql>exit
Bye

    关闭服务器:

    $ brew services stop mysql
  10. 基本完成

    其实我遇到过很多问题，也尝试解决，Google了很多，并没有很成功的解决问题，课设期间时间也比较有限，所以在尝试解决不成功后，出现问题我选择卸载，重新安装~

    根据上面打开流程，基本不会再出差啦


三、卸载MySQL

  1. 打开终端,依次输入
    $ brew remove mysql
$ brew cleanup
$ sudo rm /usr/local/mysql
$ sudo rm -rf /usr/local/var/mysql
$ sudo rm -rf /usr/local/mysql*
$ sudo rm ~/Library/LaunchAgents/homebrew.mxcl.mysql.plist
$ sudo rm -rf /Library/StartupItems/MySQLCOM
$ sudo rm -rf /Library/PreferencePanes/My*
  2. 查看是否成功
    $ mysql

    提示mysql 没有发现，多半就可以了。（其实我也不确定哈，只是我这样就可以~）

  3. 最后重启电脑重新安装即可


四、MySQL遇见mariadb(2019/03/25更)

最近尝试用homebrew 安装了mariadb，然后brew services start mriadb成功，但是运行mysql -uroot连接数据库时报错：

Can't connect to local MySQL server through socket '/tmp/mysql.sock' (2)

尝试解决未果！决定放弃mariadb，用回MySQL。故卸载mariadb，用brew services start mysql启动成功，但是mysql -uroot依旧报错；故卸载重装Mysql，用brew services start mysql启动成功，依旧连不上（尝试修改my.cnf，PATH来解决，未果）！！！！

  1. 解决办法

    • 卸载MySQL(卸载同上,甚至卸载Homebrew，重启电脑)

    • 重新下载homebrew(可跳过)

      //安装
$ /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

//卸载
$ /usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/uninstall)"

    • 下载MySQL

      $ brew install mysql

//其他常用命令
$ brew search mysql
$ brew info mysql

    • 启动MySQL

      $ mysql.server start

//其他常用命令
$ mysql.server restart //重启
$ mysql.server stop    //停止

  2. 总结

    Conflicts with:
    mariadb (because mysql, mariadb, and percona install the same binaries.)
    mariadb-connector-c (because both install plugins)
    mysql-cluster (because mysql, mariadb, and percona install the samebinaries.)
    mysql-connector-c (because both install MySQL client libraries)
    percona-server (because mysql, mariadb, and percona install the same binaries.)

由于用brew services start mysql报错，用mysql.server start正常。结合上面的Conflicts with:...because mysql, mariadb, and percona install the same binaries...,猜测，下载了mariadb的情况下，建议使用mysql.server start来启动MySQL。
