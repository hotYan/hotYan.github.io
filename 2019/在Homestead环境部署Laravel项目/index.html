在Homestead环境部署Laravel项目

  2. influxDB
  3. 任务调度(启动调度器)
  4. 队列(Supervisor配置)
  5. laravel-mix

一、安装

  1. 克隆项目

    $ git clone [项目地址]

  2. 根据部署位置拷贝对应配置文件

    $ mv .env.example .env

  3. 安装PHP库依赖

    $ sudo composer install

  4. 安装NodeJS库依赖

    $ sudo npm install

根据依赖提醒更新了包版本(可跳过)

报错如下

1.Unexpected end of JSON input while parsing near '...babel-plugin-transfor'

2.Unexpected end of JSON input while parsing near '...mjs.org/cropperjs/-/c'

解决办法

  1. 更新npm,没用

    $ npm install npm -g

  2. npm缓存问题，没用

    $ npm cache clean --force(不起作用）
$ npm cache verify（可以清除）

  3. 网络问题，解决

    $ npm config set registry http://registry.cnpmjs.org
$ npm install

    注意，package.json中的包版本应更新到较新版本。(还是不要随便更新，后面项目一直跑不起来就是更新了包版本)

二、数据库

本地安装Mariadb(不用本地数据库可跳过)

  1. 用homebrew 安装 Mariadb

    $ brew install mariadb
//其他命令
$ brew reinstall mariadb
$ brew uninstall mariadb

  2. 自启动：

    $ brew services start mariadb
//其他命令
$ brew services restart mariadb
$ brew services stop mariadb

  3. 手动启动

    $ mysql.server start
//其他命令
$ mysql.server restart
$ mysql.server stop

  4. 连接

    $ mysql -uroot


使用Homestead 自带的数据库

  1. 配置.env文件

    DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=homestead
DB_USERNAME=homestead
DB_PASSWORD=secret

  2. 使用Sequel Pro 连接数据库(配置与.env对应)

    Name: 自定义
Host: 127.0.0.1     //对应DB_HOST
Username: homestead //对应DB_USERNAME
Password: secret    //对应DB_PASSWORD
Database: 自定义
Port: 33060         //对应DB_PORT,注意要填33060

  3. 进入Homestead运行环境

    $ cd homestead
$ vagrant up
$ vagrant ssh

  4. 迁移数据表

    vagrant@homestead:~$ cd code
vagrant@homestead:~/code$ cd [project_name]   
vagrant@homestead:~/code/[project_name]$ php artisan migrate   //迁移数据表，成功后可在Sequel Pro界面看到迁移后的数据表

  5. 手动导入数据表（file -> import）

  6. 为数据表填充初始账户数据

    vagrant@homestead:~/code/[project_name]$  php artisan db:seed     //[project_name]目录下


三、InfluxDB

  1. 安装influxdb

    vagrant@homestead:~/code/[project_name]$  sudo apt-get install influxdb

  2. 启动influxDB

    vagrant@homestead:~/code/[project_name]$  influx

  3. 创建数据库[database_name]

    > create database [database_name]

  4. 查看新建数据库

    > SHOW DATABASES

  5. 退出influxDB

    > exit


四、安装opencv-python

vagrant@homestead:~/code/[project_name]$  pip install opencv-python   //Homestead环境

五、任务调度

  1. 进入Cron条目编辑状态(Homestead环境)

    vagrant@homestead:~/code/[project_name]$  crontab -e

  2. 新增Cron条目

    * * * * * php /path-to-your-project/artisan schedule:run >> /dev/null 2>&1

六、创建符号链接

vagrant@homestead:~/code/[project_name]$ php artisan storage:link

了解更多storage:link


七、配置supervisord守护进程

  1. 进入/etc/supervisor/conf.d

    vagrant@homestead: ~/code/[project_name]$  cd /etc/supervisor/conf.d
vagrant@homestead:/etc/supervisor/conf.d$  ls      //可以看到没有文件
vagrant@homestead:/etc/supervisor/conf.d$  sudo vim emr-report-queue.conf  //创建并编辑text_name.conf文本,可用【:wq /:qa!】退出

  1. text_name.conf 内容示例

    [program:emr-report-queue]
process_name=%(program_name)s_%(process_num)02d
command=php /home/vagrant/code/EmR_Web/artisan  queue:work --queue=emr-createreport--memory=100 --sleep=1 --timeout=3000 --tries=1
autostart=true
autorestart=true
user=vagrant
numprocs=1
redirect_stderr=true
stdout_logfile=/home/vagrant/code/EmR_Web/storage/logs/queue-emr-createreport.log

  2. 重新载入supervisor配置文件并重启

    vagrant@homestead:~/code/[project_name]$ sudo supervisorctl reread
vagrant@homestead:~/code/[project_name]$ sudo supervisorctl update
vagrant@homestead:~/code/[project_name]$ sudo supervisorctl start all


八、laravel-mix(本地环境)

  1. 开发环境编译前端代码(新建Teminal)

                 ~$ cd code
          code$ cd [project_name]
[project_name]$ npm run watch
[project_name]$ npm run prod


九、部署详细流程

  1. 克隆代码到code文件夹下

    $ git clone ...

    注：克隆master分支代码忽略2、3。

  2. (在项目根目录)查看远程所有分支

    $ git branch -a
//显示如下
* master
remotes/origin/HEAD -> origin/master
remotes/origin/develop
remotes/origin/master

  3. 拉取想要的分支

    $ git checkout remotes/origin/develop

  4. 将配置文件 .env 放在项目根目录

  5. (项目根目录)安装依赖

    $ composer install || sudo composer install     //安装PHP依赖
$ npm intsall || sudo npm install       //安装node依赖

  6. 修改Homestead.yaml文件

    sites:
    - map: ebfmmp.test
      to: /home/vagrant/code/FMMP_Web/public

  7. 修改/etc/hosts文件

    192.168.10.10  ebfmmp.test

  8. 进入Homestead运行环境

    $ cd Homestead
$ vagrant up || vangrant up --provision
$ vagrant ssh

  1. 配置Sequel pro 连接Homestead数据库

    DB_PORT=3306         //sequel pro一定写33060
DB_DATABASE=eb_fmmp  //自定义
DB_USERNAME=homestead 
DB_PASSWORD=secret

  2. 根据情况回滚迁移

    $ php artisan migrate:rollback

  3. 迁移数据表

    $ php artisan migrate

  1. 给数据表填充初始账户数据

    $ php artisan db:seed

  2. 建立软连接

    $ php artisan migrate:link

  3. 开发环境执行编译前端代码

    $ sudo npm run watch

  4. 生产环境执行编译前端代码

    $ sudo npm run prod

  5. 浏览器访问域名开始使用


十、结束运行

  1. 退出浏览器

  2. 退出sequel pro

  3. 退出Homestead运行环境

    $ exit
$ vagrant halt
//补充//
$ vagrant reload || vagrant reload --provision //重启
$ vagrant destory //销毁

  4. 退出命令行


十一、再次运行

  1. PULL代码

    $ git branch    //查看本地分支
$ git checkout xxx      //切换到xxx分支
$ git pull origin xxx   //拉取远程最新代码到本地xxx分支

  2. 进入Homestead运行环境

  3. 通过Sequel Pro连接数据库

  4. npm run watch //不关闭窗口

  5. 浏览器访问域名

  6. PUSH代码

    $ git status      //查看状态，即修改了什么
$ git add -A      //添加到仓库
$ git commit -m '解释说明'    //提交到仓库
$ git push origin xxx   //提交本地最新代码到远程xxx分支

十二、部署问题

  1. 报错①：

    Please provide a valid cache path.

    原因：storage文件下的framework文件缺失导致

    解决办法：添加framework文件


  1. 报错②：

    RuntimeException No application encryption key has been specified.

    解决办法：

    $ php artisan key:generate
