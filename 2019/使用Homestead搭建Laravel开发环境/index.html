使用 Homestead 搭建 Laravel 开发环境[mac][持续完善]

一、准备工作

1. 默认了解 Homestead 和 Composer.

  • 不了解Homestead？laravel homestead

  • 不了解 Composer？composer中文网




二、安装配置运行环境[mac]

1.安装 VirtualBox

  • 选择适合自己的版本下载


2. 安装 Vagrant

  • 选择适合自己的版本下载

  • 检测是否成功

    $ vagrant -v    //任意目录
Vagrant 2.2.3   //出现版本信息，成功


3. 安装 Homestead Vagrant Box

  • 在线安装(非常耗时，请挂VPN，网速不行的直接选择离线安装)

    $ vagrant box add laravel/homestead     //根目录

  • 离线安装

    该链接附录一写的很清楚，不赘述.


4. 安装 Homestead

  • 从git克隆homestead项目

    $ cd XX   //进入到安装Homestead的目录，可以就在当前根目录下安装Homestead
$ git clone https://github.com/laravel/homestead.git Homestead    //从git克隆homestead项目

  • 创建配置文件

    $ cd Homestead  //进入Homestead目录
$ bash init.sh  //创建配置文件


5. 编辑 Homestead.yaml 文件（根据需要添加）

  • ip(后面需要)

    ip: "192.168.10.10"

  • authorize & keys (后面需要)

    authorize: ~/.ssh/id_rsa.pub
keys: - ~/.ssh/id_rsa

    如果之前电脑没有添加过SSH 秘钥，可参考文章Github添加SSH密钥

  • provider(在安装中选3即可)

    provider: virtualbox

  • folders(共享文件)

    folders:
    - map: ~/code/hotyan
    to: /home/vagrant/code/hotyan

    需要在当前目录新建code文件夹

    $ mkdir -p code //用命令行创建

  • site(配置Nginx站点)

    sites:
    - map: homestead.test   //站点名称可修改，以.test结尾即可(.app不可用，被google买了)
    to: /home/vagrant/code/hotyan/public    //对应public下的index.php,可省略


6. 编辑 /etc/hosts 文件

  • 修改/etc/hosts 的权限为可写

  • 方法一:

    • command + shift + G

    • 进入/etc/hosts

    • 编辑/etc/hosts

      192.168.10.10  homestead.test   //添加到hosts文件尾

  • 方法二：

    • cd /etc //进入/etc目录

    • vi hosts // 修改hosts文件

    • 按 i 进入编辑状态

    • 按esc 退出编辑状态，输入 :wq 保存退出


7. 启动虚拟机（Homestead目录）

$ vagrant up    //，直接启动
$ vagrant up --provision    //若Homestead.yml文件有改动,加上--provision


8. SSH 登入虚拟机

$ vagrant ssh


9. 退出并关闭虚拟机

$ exit    //退出
$ vagrant halt    //关闭虚拟机

///////补充/////////
$ vagrant reload --provision    //重启虚拟机
$ vagrant destroy   //销毁虚拟机


10. 用 Composer 创建 Laravel 项目

$ cd code
$ composer create-project laravel/laravel hotyan --prefer-dist

–prefer-dist: 意思是下载用于 distribution 的稳定版本，而不是开发中的版本。


11. 在浏览器输入homestead.test




三、安装composer

1. 下载composer.phar 文件

  • 下载地址


2. 全局安装composer

$ sudo mv composer.phar /usr/loacl/bin/composer     //将composer.phar文件拷贝到/usr/loacl/bin/目录

$ composer      //查看是否成功

//查看权限
$ cd /usr/local/bin/    
$ ls -al

//修改权限
$ sudo chmod -R 755 composer

$ composer      //查看是否成功


3. composer 中国镜像

//查看镜像
$ composer config -gl repo.packagist

//配置为中国镜像
$ composer config -g repo.packagist composer https://packagist.phpcomposer.com

//再次查看镜像
$ composer config -gl repo.packagist


4. 更新composer(v1.8.0 => v1.8.4)

$ composer self-update


4.1. composer更新到1.8.4失败

运行vagrant up 时，出现如下提示：

homestead-7:   [ErrorException]              
homestead-7:   rename(/home/vagrant/.composer/cache/composer-temp.phar,/usr/local/bin/composer): Permission denied

原因：homestead v8.0.1 的 bug，已在v8.0.2修复，故更新homstead到v8.0.2 。(更新后，查看composer版本为1.8.4，问题解决。)


4.2.更新Vagrant box(v7.0.0 => v7.1.0)

  1. 删除虚拟机

    $ cd Homestead  //所有命令都在此目录下进行
$ vagrant destroy


  1. 更新Laravel/Homestead Vagrant box(v7.0.0 => v7.1.0)

    $ vagrant box update //不是一次性成功

问题1(暂未明白)

OpenSSL SSL_read: SSL_ERROR_SYSCALL, errno 54

问题2(暂未明白)

Failed to connect to vagrantcloud-files-production.s3.amazonaws.com port 443: Operation timed out

多尝试几次，vagrant box 已更新到v7.1.0。


  1. 删除旧版本

    $ vagrent box remove --box-version 7.0.0


4.3 更新homestead(v8.0.1 =>v 8.0.2)

  1. 更新Homestead源码

    如果通过克隆git repo 安装的Homestead源代码，运行：

    $ git pull origin master

    master可能不稳定，保证处于稳定版本，跟最新的稳定版本进行比较：

    $ git checkout v8.0.2


  1. 创建新的虚拟机

    $ vagrant up

    composer更新到1.8.4成功，问题解决。
