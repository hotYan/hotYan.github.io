一、Git

  1. 查看分支

    $ git branch      //查看本地
$ git branch -r   //查看远程
$ git branch -a   //查看全部

  2. 新建分支

    $ git branch xxx

  1. 切换分支

    $ git checkout xxx

  2. 新建并切换分支

    $ git checkout -b xxx

  3. 删除分支

    $ git branch -d xxx     //删除本地分支
$ git push origin :xxx     //删除远程分支
$ git push origin xxx     //提交分支

  4. 拉取远程最新代码

    $ git pull origin xxx   //xxx为分支，可master可任意

  5. 提交本地最新代码

    $ git status      //查看状态，即修改了什么
$ git add -A      //添加所有修改内容到仓库
$ git commit -m "解释说明"    //提交内容到仓库，并解释说明
$ git push origin xxx   //提交到远程xxx分支

  6. 查看提交历史

    $ git log 
$ git log -p <file> //产看指定文件的提交历史

  7. 本地项目第一次上传远程空仓库

    $ git init  //确保本地项目是git项目
$ git remote add origin xxxxx   //关联远程仓库

$ git add .
$ git commit -m xxxx

$ git push -u origin master  //关联远程分支并提交代码到该分支

$ git remote remove origin      //取消远程连接


二、终端

  1. 目录

    $ cd  
$ ls 
$ ls -la   
$ pwd   //打印当前目录
$ mkdir xxx //创建目录
$ rmdir xxx     //删除目录
$ mvdir xxx

  2. 文件

    $ open xxx  //使用默认程序打开
$ cp    //复制
$ vim name.text //新增修改
$ rm    //删除文件
$ rm -r //删除目录
$ mv    //改变


三、Git 提交注释

  1. [feat]:新功能

  2. [fix]:修复bug

  3. [add]:添加某些内容

  4. [remove]:删除某些内容

  5. [update]:更新某些内容

  6. [refactor]:重构(不是新功能、修复bug)

  7. [style]:格式

  8. [chore]:琐事

  9. [docs]:文档

  10. [test]:测试


四、SSH 连接远程服务器

连接服务器需要知道服务器IP、端口号、服务器的账号和密码:

  1. 打开终端,进入根目录

    $ sudo su -             //进入root目录
$ password: xxxxx       //输入电脑账户密码

  2. 连接

    $ ssh -p 端口号 用户名@IP       //如 ssh -p 19222 web@10.1.63.167
$ yes                         // 确认连接
$ password: xxxxx             //输入服务器账户密码

  3. 连接成功

  4. git pull

  5. npm run dev


五、Docker

  1. 查看docker信息

    $ docker info

  2. 搜索镜像

    $ docker search [OPTIONS] TERM

  3. 下载镜像

    $ docker pull [OPTIONS] NAME    //下载镜像

  4. 提交镜像

    $ docker login
$ docker push [OPTIONS] NAME[:TAG]    //提交镜像

  5. 查看镜像

    $ docker images [OPTIONS]   //Usage
$ docker images -a          //查看全部镜像

  6. 查看容器

    $ docker ps [OPTIONS]   //Usage
$ docker ps -a          //查看全部容器

  7. 删除镜像

    $ docker rmi [OPTIONS] IMAGE [IMAGE...]     //Usage
$ docker rmi IMAGE       //删除镜像

  8. 删除容器

    $ docker rm [OPTIONS] CONTAINER [CONTAINER...]  //Usage
$ docker rm CONTAINER     //删除容器

  9. 运行镜像

    $ docker run --help 

# Usage
$ docker run [OPTIONS] IMAGE [COMMAND]  //Run a command in a new container

# 参考
$ docker run IMAGE      //前台挂起，如hello-world
$ docker run -d IMAGE   //后台运行
$ docker run
    --name [NAME]       //Assign a name to the container
    -v /home/Docker:/hotyan//映射到宿主机目录/home/Docker
    -p 8888:80           //端口映射
    -d                   //Run container in background and print container ID
    IMAGE                //镜像名

  10. [启动|暂停|重启]容器

    $ docker start [OPTIONS] CONTAINER [CONTAINER...]
$ docker start CONTAINER

$ docker stop [OPTIONS] CONTAINER [CONTAINER...]
$ docker stop CONTAINER

$ docker restart [OPTIONS] CONTAINER [CONTAINER...]
$ docker restart CONTAINER

  1. 进入容器

    $ docker exec --help
# Usage
$ docker exec [OPTIONS] CONTAINER COMMAND   //Run a command in a running container

$ docker exec 
    -it //-i & -t   //OPTIONS
    CONTAINER
    bash    //COMMAND

  2. 网络

    $ docker network COMMAND
Commands:
    connect     //Connect a container to a network
    create      //Create a network
    disconnect  //Disconnect a container from a network
    inspect     //Display detailed information on one or more networks
    ls          //List networks
    prune       //Remove all unused networks
    rm          //Remove one or more networks

  3. 查看hosts

    $ cat /etc/hosts

  4. docker-compose

    // docker-compose.yml同级目录运行
$ docker-compose build          // Build or rebuild services
$ docker-compose up -d          // up: Create and start containers, -d: Run containers in the background
$ docker-compose stop           //Stop running containers without removing them.
$ docker-compose start          //Start existing containers.
$ docker-compose restart        //Restart running containers.

  5. 启动基于 docker 的 hyperf + mysql + redis 运行环境

    //1. 已存在项目
$ git clone {Boomdocker-dev}
$ cd boomdocker-dev

//构建项目
$ docker-compose build  
=> boomdocker-dev 

//新建并后台运行容器
$ docker-compose up -d          //同级会生成一个project目录，存放hyperf项目代码
//将后端代码clone到project目录下
$ git clone xxx -b develop
//配置后端代码的env
DB_DATABASE=hyperf
DB_PASSWORD=boom123456

//以bash方式进入hyperf容器
$ docker exec -it boomdock-dev_hyperf_1 bash
//安装依赖
$ compose install 
//第一次需要初始化数据,进入容器项目目录,初始化数据
$ php bin/hyperf.php migrate
$ php bin/hyperf.php db:seed
//运行监听项目
$ php watch

//运行前端项目
$ npm run serve  //=> 访问 http://localhost:9502

//关闭容器
$ docker-compose stop 
//第二次运行
$ docker-compose start

//更新数据库
$ php bin/hyperf.php migrate:fresh
$ php bin/hyperf.php db:seed

  5. Node.path

    const path = require('path')
//join():把path片段连接到一起，并规范化生成的路径
const src1 = path.join(__dirname,'/a')     //=>/User/hotyan/test/a
const src2 = path.join(__dirname,'/a/b')   //=>/User/hotyan/test/a/b
const src3 = path.join('/a','b','/c/d','e','..')    //=>/a/b/c/d
const src4 = path.join('a','b','c')                 //=>a/b/c
const src5 = path.join('a','b','..','..')           //'.',当前工作目录

//resolve():把一个路径或路径片段的序列解析为一个绝对路径
const src6 = path.resolve('/a/b','/c/d')    //=>/c/d
const src7 = path.resolve('/a/b','c/d')     //=>/a/b/c/d
const src8 = path.resolve('/a/b','../c/d')  //=>/a/c/d
const src9 = path.resolve('a','b')          //=>/User/hotyan/test/a/b
