第五周

什么是浏览器同源策略？我们在日常开发调试时有无跨域？如有，是如何跨域的？

  1. 同源策略&跨域

    浏览器的同源策略导致了跨域。

  2. 同源策略

    限制了从同一个源加载的文档或脚本如何与来自另一个源的资源进行交互。是一个用于隔离潜在恶意文件的重要安全机制,即保证用户信息安全，防止恶意的网站窃取数据，如跨站请求伪造(CSRF)攻击。

  3. 同源

    • 协议相同
    • 域名相同
    • 端口相同

  4. 非同源，三种行为受到限制

    • Cookie、LocalStorage、IndexDB 无法读取
    • DOM无法获得
    • AJAX请求不能发送

  5. 解决跨域

    • 跨域资源共享（Cross-Origin Resource Sharing）
    • JSONP
    • 服务器代理
    • document.admin
    • postMessage
    • location.hash
    • window.name
    • 图像Ping
5.1 CORS

浏览器将CORS请求分为：`简单请求`和`非简单请求`,同时满足一下两大条件就属于简单请求：
- ① 请求是 HEAD、GET、POST 三种方法之一；
- ② HTTP头部信息不包含除 Accept、Accept-Language、Content-Language、Last-Event_ID、Content-Type以外的字段

不满足则属于非简单请求；

<br/>

5.2 JSONP

- JSONP 跨域的基本原理:由于 script 标签不受浏览器同源策略的影响，允许跨域引用资源。因此可以通过动态创建 script 标签，然后利用 src 属性进行跨域。

第一周

请描述浏览器输入URL到页面展示的整个过程??

  1. 流程概览
    • 将域名解析成IP地址(DNS 解析)
      • 输入URL地址,浏览器从url中解析出主机名、端口号
      • 浏览器将主机名转换成服务器ip地址（浏览器先查找本地DNS缓存列表,没有的话,再向浏览器默认的DNS服务器发送查询请求,同时缓存）
    • 浏览器建立一条与目标Web服务器的TCP连接(TCP 三次握手)
    • 浏览器向服务器发送一条HTTP请求报文
    • 服务器处理请求并向浏览器返回一条HTTP响应报文
    • 浏览器解析渲染绘制页面,浏览器可能会重复发送http请求,直至资源全部加载完毕(浏览器渲染机制）
    • 断开连接(TCP 四次挥手)

  1. 浏览器渲染机制

    ① 根据 HTML 解析出 DOM 树

    • 解析HTML构建成一个DOM树(Document Object Model,文档对象模型），DOM 树的构建过程是一个深度优先遍历过程：当前节点的所有子节点都构建好后才会去构建当前节点的下一个兄弟节点。

    • 若遇到 script 标签，则 DOM 树的构建会暂停，直至脚本执行完毕。


      ② 根据 CSS 解析生成 CSS规则树(CSS Rule Tree)

    • 解析 CSS规则树 时 js执行 将暂停，直至 CSS规则树 就绪。

    • 浏览器在 CSS规则树 生成之前不会进行渲染。


      ③ 根据 DOM树 和 CSSOM 来构造 Rendering Tree（渲染树）

    • DOM树 和 CSS规则树 全部准备好了以后，浏览器才会开始构建 渲染树。

    • 精简CSS 并可以加快 CSS规则树的构建，从而加快页面相应速度。

      注意： Rendering Tree 渲染树并不等同于 DOM 树，因为一些像 Header 或 display:none 的东西就没必要放在渲染树中了。


      ④ 根据渲染树计算每一个节点的信息

      有了Render Tree，浏览器已经能知道网页中有哪些节点、各个节点的CSS定义以及他们的从属关系。下一步操作称之为Layout，顾名思义就是计算出每个节点在屏幕中的位置 layout render tree：

    • 布局（Layout）：通过渲染树中渲染对象的信息，计算出每一个渲染对象的位置和尺寸

    • 回流：在布局完成后，发现了某个部分发生了变化影响了布局，那就需要倒回去重新渲染。


      ⑤ 根据计算好的信息绘制页面

      绘制(Paint)，即遍历render树，并使用浏览器UI后端层绘制每个节点：

    • 绘制阶段，系统会遍历呈现树(render tree)，并调用呈现器的“paint”方法，将呈现器的内容显示在屏幕上。

    • 重绘：某个元素的背景颜色，文字颜色等，不影响元素周围或内部布局的属性，将只会引起浏览器的重绘。


  1. 重绘 & 回流

    重绘不一定回流，回流一定重绘。

    • 重绘： 不影响周围布局，修改元素背景颜色、文字颜色等行为。
    • 回流：元素尺寸发生了变化，需重新计算渲染树，重新渲染的行为。
